TokenSage

Hybrid Token-Efficient Routing Agent

Minimize Fireworks AI token spend by routing tasks between a free local vLLM model and a paid large model — without sacrificing accuracy.

Problem

The Token Efficiency Challenge

Local inference is free, but limited

AMD MI300X runs small models at zero cost, but a 3B model lacks accuracy on complex tasks.

Every Fireworks token costs you

The leaderboard ranks by token spend. The winner is whoever uses the fewest Fireworks tokens while meeting the accuracy gate.

The routing tradeoff

Route too aggressively and accuracy drops. Route too conservatively and the token budget burns. You need a smart, adaptive router.

Architecture

Routing Pipeline

1 Router Qwen 3B classifies task type and decides LOCAL or BIG with confidence
2 Local Solver Qwen 3B produces a structured JSON answer (thinking, answer, confidence)
3 Self-Verification Re-answers from scratch without seeing the prior output; compares both
4 Retry or Escalate If verification fails, retry locally with chain-of-thought; if still fails, escalate to Fireworks 72B

No Fireworks key? Falls back to local vLLM Qwen 14B automatically.

Core Innovation

Two-Stage LLM Router

Classifies tasks into 5 types, then applies few-shot examples to decide LOCAL vs BIG. Low-confidence LOCAL decisions (<0.3) auto-escalate to BIG.

Key Technique

Re-Answer Verification

Instead of asking “Is this correct?” — which small models confidently fail at — we ask the model to solve from scratch again.

Naive approach

Asks the model to judge its own answer — suffers from confirmation bias

Small models almost never admit mistakes

TokenSage approach

Re-answers independently without seeing the original output

Compares both results: if they differ, the original was likely wrong

If the verifier’s confidence is below 0.5, we treat it as a failure regardless of answer match.
