TokenSage

Hybrid Token-Efficient Routing Agent

Minimize Fireworks AI token spend by routing tasks between a free local vLLM model and a paid large model — without sacrificing accuracy.

Problem

The Token Efficiency Challenge

Local inference is free, but limited

AMD MI300X runs small models at zero cost, but a 3B model lacks accuracy on complex tasks.

Every Fireworks token costs you

The leaderboard ranks by token spend. The winner is whoever uses the fewest Fireworks tokens while meeting the accuracy gate.

The routing tradeoff

Route too aggressively and accuracy drops. Route too conservatively and the token budget burns. You need a smart, adaptive router.

Architecture

Routing Pipeline

1 Deterministic 11 regex/rule solvers catch pure math, date math, unit conversions, sentiment, JSON extraction — 0 tokens
2 Classifier Regex-based task categorizer maps prompts to 8 types (math, code, logic, sentiment, NER, summary, factual, extract)
3 Local Ensemble Qwen 3B at 3 temperatures [0.0, 0.3, 0.7]; 2/3+ majority vote returns with 67%+ confidence
4 Category Solve Single Qwen 3B call with category-specific system prompt and temperature from prompts.py
5 Fireworks API Paid 72B call with auto-retry on failure; only reached when all local options are exhausted

All local steps (1-4) cost zero Fireworks tokens. The ensemble alone adds 3 free calls per task.

Key Technique

Multi-Temperature Ensemble

Instead of asking the model “Is this correct?” — which small models confidently get wrong — we call it 3 times at different temperatures and look for a consensus.

Single-call approach

One deterministic pass — brittle, easy to fall into local minima

One unlucky token sample and the answer is wrong

TokenSage ensemble

3 parallel vLLM generations at temps 0.0, 0.3, 0.7 — batched in one GPU call

2/3+ majority vote signals high confidence; disagreement means uncertainty

If the ensemble doesn’t converge, the task falls through to Fireworks — no bad answers slip past.
